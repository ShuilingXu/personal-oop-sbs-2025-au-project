
## 课程项目描述：虚拟团队项目管理系统

**项目目标**：
本项目要求学生基于面向对象的设计思想，开发一个**控制台的虚拟团队项目管理系统**。
系统中将包含**不同类型的项目任务**（如功能开发、缺陷修复），并管理**项目、团队成员和任务**之间的复杂关系。
学生需要应用**类和对象、继承、多态、封装**等核心面向对象概念，并特别利用**泛型**和**集合框架**来组织项目结构。
**多线程、数据库操作和文件流**的应用将作为高级功能要求实现。
本项目使用控制台交互，强化程序的逻辑设计和面向对象编程概念的应用。

-----

#### 1\. **功能要求**

* **用户注册和登录**：
    * 系统需要包含两个用户角色：**普通成员（User）和项目经理（Admin）**。
    * 普通成员可以**查看、更新自己被分配的任务**，并查看项目总体信息。
    * 项目经理可以**创建/删除项目、增删成员、创建/分配任务**。
* **项目和成员管理**：
    * **项目创建/删除**：项目经理可以创建新的项目，并为其设置名称、描述、截止日期。
    * **成员管理**：项目经理可以添加/移除团队成员到特定项目。
    * **成员信息**：存储成员的基本信息（ID、姓名、角色）。
* **任务管理 (核心 OOP 考察)**：
    * **任务添加**：能够添加新的任务，包括**通用信息**（如任务标题、描述、优先级、状态）和**特定类别信息**。
    * **任务类型**：至少设计两种任务类型：`FeatureTask`（功能开发任务）和 `BugReport`（缺陷报告）。
    * **任务分配**：项目经理可以将任务分配给项目内的特定成员。
    * **任务更新**：成员可以更新自己任务的状态（待办、进行中、已完成）和进度。
    * **任务查询/筛选**：支持按**成员、项目、任务状态或优先级**进行筛选查询。
* **面向对象特性要求**：
    * **类和对象**：创建抽象基类 `TaskItem`，以便支持不同类型任务的扩展。核心类包括 `Project`、`TeamMember`、`TaskItem`。
    * **继承**：设计子类 `FeatureTask` 和 `BugReport` 继承自 `TaskItem`。子类需新增特定属性（如 `BugReport` 需有严重程度 `Severity`）。
    * **封装**：所有类的属性需使用封装。
    * **多态**：在父类中定义**展示任务详情** (`displayDetails()`) 的方法，要求子类根据自身类型输出特定的信息（例如，`BugReport` 需要突出显示其严重程度）。
* **泛型、集合与数据结构**：
    * 使用 **Java 集合框架**来存储和管理项目、成员和任务。
    * 项目内部结构需使用**嵌套集合**来体现复杂关系，例如：
        * 存储所有项目：`Map<String, Project>` (Key: ProjectName)
        * 在 `Project` 类内部，存储成员和任务分配：`Map<TeamMember, List<TaskItem>>`。
    * 设计一个**泛型**类或方法（如 `TaskProcessor<T extends TaskItem>`），用于处理和统计不同类型任务的集合。

#### 2\. 交互方式

* **命令行菜单**：创建一个简洁的控制台菜单，支持用户输入数字选择操作。

  ```
  Welcome to the Team Project Management System
  1. Register
  2. Login
  3. View Assigned Tasks (User)
  4. Search Tasks by Project/Status
  5. Admin Menu (Project Manager Only)
  6. Logout
  7. Exit
  Please enter your choice: _
  ```

* **Admin Menu 示例：**

  ```
  Project Manager Menu
  1. Create New Project
  2. Add Team Member
  3. Create New Task (Assign to Member)
  4. View All Projects & Status
  5. Run Progress Report (Advanced)
  6. Back to Main Menu
  Please enter your choice: _
  ```

#### 3\. **高级功能要求 (加分项)**

* **数据库支持**：
    * **数据持久化**：项目、成员、任务数据以及它们之间的关系应存储在数据库中（MySQL 或 SQLite）。
    * **技术要求**：使用 **JDBC** 进行数据库连接、数据增删查改（CRUD 操作）。
* **多线程**：
    * 实现一个**后台线程** (`ProgressReporterThread`)，用于**定期生成项目进度报告**。
    * 该线程应每隔一段时间（如 30 秒）运行一次，计算所有项目当前任务的完成率，并记录到日志文件或控制台，不影响用户的正常操作。
* **文件存储与读取**：
    * 使用文件流将多线程生成的**项目进度报告**或**系统操作日志**写入文件，并支持管理员读取这些历史报告的功能。

#### 4\. **评分标准**

* **功能完整性（50%）**：
    * 基本功能（30%）：用户登录/权限，项目、成员、任务的增删查改功能，任务分配与状态更新，控制台界面，**泛型和复杂集合（嵌套 Map）的正确应用**。
    * 高级功能（20%）：多线程进度报告、文件日志存储和读取、数据库操作。
* **面向对象设计（30%）**：
    * 类的设计合理性（Project, TeamMember, TaskItem 等）、继承关系、封装性、多态性的正确使用。
* **代码质量（20%）**：
    * 代码结构清晰，命名规范，注释完整，代码风格符合规范。
